JS Runtime & V8 Engine

Як працює JavaScript?

JavaScript — це інтерпретована (або JIT-компільована) мова. Найпопулярніший рушій — V8 (Chrome, Node.js). Він перетворює JS-код безпосередньо в машинний код для швидкодії.

Memory Heap: де зберігаються об'єкти.
Call Stack: де виконуються функції.
Global Info:
  • Single-threaded: один потік виконання.
  • Non-blocking: асинхронність через Event Loop.
  • JIT Compilation: компиляція "на льоту".

JavaScript Basics

Типи даних

У JavaScript є 8 основних типів даних:

  • Primitives: string, number, bigint, boolean, undefined, symbol, null.
  • Objects: Об'єкти, масиви, функції.
BigInt: Для роботи з цілими числами довільної довжини. const bigValue = 9007199254740991n;
Symbol: Унікальні та незмінні ідентифікатори. const sysID = Symbol("id"); 
typeof null; // "object" (bug)
typeof undefined; // "undefined"
typeof function(){} // "function"

var vs let vs const

Ключова різниця полягає у Scope (області видимості) та Hoisting (піднятті).

Особливість var let const
Scope Function Block { } Block { }
Hoisting Так (undefined) Так (TDZ*) Так (TDZ*)
Reassign Так Так Ні

*TDZ (Temporal Dead Zone): період від початку блоку до оголошення змінної, коли доступ до неї заборонений.

Hoisting Example:
console.log(a); // undefined
var a = 10;

console.log(b); // ReferenceError
let b = 20;
Scope Example:
if(true) {
    var x = 1;
    let y = 2;
}
console.log(x); // 1
console.log(y); // ReferenceError

Common Array Methods

forEach vs map

forEach просто ітерує, map створює новий масив. 

const nums = [1, 2, 3];
nums.forEach(n => console.log(n));

const squared = nums.map(n => n * n); // [1, 4, 9]
filter & sort

filter відсіює елементи. sort мутує оригінальний масив!

const filtered = [1, 5, 10].filter(n => n > 4); // [5, 10]

const sorted = [10, 2, 5].sort((a, b) => a - b); // [2, 5, 10]
reduce

Зводить весь масив до одного значення (сума, об'єкт тощо).

const sum = [1, 2, 3, 4].reduce((acc, curr) => acc + curr, 0); // 10
acc — акумулятор (накопичений результат), curr — поточний елемент.

Scopes & Closures

Замикання (Closure)

Це функція, яка "пам'ятає" своє лексичне оточення, навіть коли вона виконується поза ним.

  • Навіщо: Приватні змінні, інкапсуляція даних, фабрики функцій.
function createCounter() {
    let count = 0;
    return () => ++count;
}
const counter = createCounter();
counter(); // 1
counter(); // 2

Asynchronous JavaScript

Promises (Проміси)

Об'єкт, що представляє результат асинхронної операції, який буде доступний у майбутньому.

Стани (States):
  • Pending: очікування.
  • Fulfilled: успішно (викликано resolve).
  • Rejected: помилка (викликано reject).
Створення промісу (Constructor):
const myPromise = new Promise((resolve, reject) => {
    const success = true;
    if (success) resolve("Дані отримано!");
    else reject("Помилка запиту.");
});
Методи для групи промісів:
Метод Опис
Promise.all([p1, p2]) Чекає на всі. Якщо хоч один впаде — падає весь.
Promise.allSettled([p1, p2]) Чекає на завершення всіх (успіх або помилка).
Promise.race([p1, p2]) Повертає результат першого завершеного (будь-якого).
Promise.any([p1, p2]) Чекає на перший успішний.
Приклад обробки ланцюжком (Chaining):
fetchData()
    .then(data => process(data))    // викликається при успіху
    .catch(error => console(error)) // викликається при помилці
    .finally(() => stopLoader());   // в будь-якому випадку

Async / Await

Синтаксичний цукор над промісами, що дозволяє писати асинхронний код як синхронний.

async function getData() {
    try {
        const data = await fetch('url');
        return await data.json();
    } catch (err) {
        // handle error
    }
}

Event Loop

Механізм, що дозволяє JS виконувати неблокуючі операції, попри те, що він однопотоковий.

  • Call Stack: де виконується код.
  • Microtasks: (Promises) мають пріоритет.
  • Macrotasks: (setTimeout, I/O).

Objects & Prototypes

Прототипне успадкування

У JavaScript успадкування працює через об'єкти-прототипи. Кожен об'єкт має посилання на свій прототип (__proto__).

const animal = { eats: true };
const rabbit = Object.create(animal);
console.log(rabbit.eats); // true (взято з прототипу)
Класи в ES6: Це лише "синтаксичний цукор" над прототипами, вони не створюють нову модель успадкування.

TypeScript Overview

Що таке TypeScript?

Це сувора надбудова (superset) над JavaScript, яка додає статичну типізацію. Код TS компілюється в чистий JS.

1. Базові типи
let isDone: boolean = false;
let count: number = 42;
let name: string = "User";
let list: number[] = [1, 2, 3];
let tuple: [string, number] = ["hello", 10];
2. Interfaces & Types
interface User {
    id: number;
    name: string;
    role?: string; // Optional property
}
const user: User = { id: 1, name: "Admin" };
3. Enums
enum Status { 
    Active, 
    Inactive, 
    Pending 
}
let current: Status = Status.Active;
4. Generics (Узагальнення)
function identity(arg: T): T {
    return arg;
}
let output = identity("myString");
Union & Intersection Types

Union (|): Змінна може бути одного з кількох типів.

let padding: string | number;

Intersection (&): Об'єднання кількох типів в один.

type Admin = User & { permissions: string[] };
Корисні Utility Types
  • Partial<T> — робить всі поля опціональними.
  • Readonly<T> — робить об'єкт незмінним.
  • Pick<T, K> — обирає лише вказані поля.
  • Omit<T, K> — видаляє вказані поля.
type UserPreview = Pick;
const preview: UserPreview = { id: 2, name: "Guest" };
